## Strategy evaluation

| Strategy | Verdict | Reason |
|----------|---------|--------|
| **A — Git submodule** | **Deferred** | No approved Fuzor Git remote exists; cannot create remote or submodule URL |
| **B — Private registry** | **Rejected** | No registry namespace, credentials, or Vercel token configuration |
| **C — Git dependencies** | **Rejected** | Requires same missing Fuzor remote; pnpm subdirectory Git deps not verified |
| **D — Committed artifacts** | **Selected** | Deterministic, Vercel-accessible, no credentials, supports clean checkout today |

Strategy D is explicitly classified as an **interim distribution mechanism**. When a Fuzor GitHub repository is approved, migrate to Strategy A (submodule) without changing runtime wrappers.
